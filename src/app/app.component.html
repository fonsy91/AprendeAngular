1. ESTRUCTURA Y MODIFICACION DE LA APP.

La diferencia entre una pagina web convencional y una single page application (SPA) estriba en que la primera esta haciendo peticiones al servidor continuamente cada vez que queremos ver una imagen se le hace una peticion al servidor y este nos devuelve otro html con esa imagen, en cambio para una SPA esto no es asi esta realiza una sola peticion al arranque de la aplicacion. El cliente va modificando cosas en la aplicacion y van apareciendo otras esto se hace gracias a la tecnologia AJAX es decir cada vez que el usuario realiza una accion se envia una peticion al servidor y este nos responde con un documento JSON se usa para intercambiar datos entre el cliente y el servidor podemos apreciarlo en la imagen de la izquierda.

A continuacion vemos el modulo raiz app.module.ts y dentro se define el componente principal a cargar AppComponet. 2. COMPONENTES EN ANGULAR.

Los componentes se pueden crear manualmente o automaticamente a partir del CLI. Una vez que creas un componente es necesario registrarlo en la aplicacion dentro del archivo app.module.ts agregamos dentro del apartado declarations el nombre de la clase del nuevo componente ademas de importar el componente con import.

Muchas veces cuando creamos un componente Angular nos crea los tres archivos primordiales html, ts y css pero algunas veces el codigo del ts y css son muy pequeños y no queremos esos archivos para algo tan minusculo es por ello que podemos hacerlo inline para ello debemos ir al archivo ts del componente omitir la linea de css o ts del @Componet y cambiamos el codigo styleUrls: ['./empleado.component.css'] por styles:[""] igual que en la programacion web que puedes hacer estilos en la misma pagina o en un archivo aparte y de la misma manera lo hariamos para la parte del html. Si quieres que al crear un componente mediante CLI haga todo esto que acabamos de comentar se deberia crear de la siguiente manera: ng g c nombreComponeten -s -t

4. PROPERTY BINDING.

Binding es poder hacer una union entre una propiedad de la parte logica con un elemento del html ya sea un cuadro de texto, una frase, un input, un boton etc.. en este caso tenemos deshabilitado el campo de introducir una empresa con la propiedad de disabled si creamos una variable como pude ser habilitaCuadro=true; en nuestra parte logica y queremos cambiar indistintamente de true a false esta propiedad dependiendo de los factores que sean como por ejemplo que algunos usuarios no tienen acceso a ese campo pues mediante un if y esta propiedad podemos hacer que se habilite o no, en el html se realiza de la siguiente manera: [disabled]="habilitaCuadro", En el aso del boton esta deshabilitado ya que hemos usado esta misma tecnica para cancelarlo por el momento.


Otro ejemplo con otro elemento de html un checkbox con esto haremos que cuando arranque la aplicacion este elemento este o no este activado dependiendo de la funcion cambiaCheck() si la tenemos a true o a false implementada en la el .ts. 5. BINDING BIDIRECCIONAL

Como vimos anteriormente en las explicaciones de la propiedad binding en ese caso cuando cambiabas una propiedad en el .ts esta se modificaba en el html esto implicaba una sola direccion, y en el event binding se hacia lo mismo pero atrapando el evento y a traves de una funcion cuando el usuario interactuaba con la aplicacion o el tempalte eso tenia reflejo en el .ts ejecutandose dicha funcion en este caso el flujo es contrario al del binding, cuando el flujo va tanto de un lado para otro se llama binding bidireccional o two way binding, esto nos permite actualizaciones continuas es decir modificar cosas en nel .ts a partir de accione en html y estas .ts se modifican al instante en el html otra vez. Para poder hacer todo esto necesitamos: la directiva ngModel y el uso de Banana in Box que es una sintaxis si recordais para el property banding usabamos [] para el flujo .ts hacia html y para el event binding () flujo de html a .ts, para este caso que queremos que sea bidireccional tendremos que suar ambos simbolos [()]. El ejemplo ilustrativo se veria en la parte superior donde usamos todos los tipos de bindings. Como podemos ver en el campo de testo aparece el nombre de Google esto es debido a que al usar el ngModel [(ngModel)]="empresa2" viaja del .ts al html y cuando escribo ocurre de forma diferente. Para poder usar todo esto en angular es necesario importarlo en el modulo principal app.module.ts en el apartado de imports FormaModule.
